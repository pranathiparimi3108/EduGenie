# Security Policy

## Supported Versions

The following versions of EduGenie are currently supported with security updates.

| Version | Supported |
|----------|-----------|
| 1.0.x    | ✅ Yes |
| < 1.0    | ❌ No |

---

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

- Do **not** disclose security vulnerabilities publicly through GitHub Issues.
- Contact the project maintainer with complete details of the vulnerability.
- Include steps to reproduce the issue, if possible.
- Valid security reports will be reviewed and addressed as quickly as possible.

---

## Security Best Practices

To ensure the security of the EduGenie project, follow these guidelines:

- Never commit `.env` files to the repository.
- Never expose your `GEMINI_API_KEY` or any API credentials in the source code.
- Store all sensitive information using environment variables.
- Rotate API keys immediately if they are accidentally exposed.
- Validate all user inputs before processing.
- Keep project dependencies updated to avoid known vulnerabilities.
- Use HTTPS when deploying the application online.
